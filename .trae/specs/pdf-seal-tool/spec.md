# PDF 图片印章工具规范

## 为什么需要此功能

当前需要实现一个工具，可以对 PDF 文件批量添加图片印章，满足合同盖章、文档认证等场景需求。

## 变更内容

- 创建 Python 项目框架
- 实现 PDF 读取和页面处理功能
- 实现图片印章加载和尺寸控制
- 实现印章位置控制
- 实现命令行接口

## 影响范围

- 受影响的功能：PDF 批量盖章
- 受影响的代码：主程序模块、CLI 接口

## 新增需求

### 需求：PDF 图片印章功能

系统应提供通过命令行对 PDF 文件添加图片印章的功能。

#### 场景：成功添加印章

- **当** 用户提供有效的 PDF 文件、图片印章、尺寸和位置参数
- **则** 生成带有印章的新 PDF 文件

#### 场景：参数验证失败

- **当** 用户提供的参数无效（文件不存在、尺寸为负数等）
- **则** 显示友好的错误信息并退出

### 需求：命令行参数

系统应支持以下命令行参数：

| 参数 | 必填 | 说明 | 格式 |
|------|------|------|------|
| --pdf, -p | 是 | PDF 文件路径 | 文件路径 |
| --image, -i | 是 | 印章图片路径 | 文件路径 |
| --width | 是 | 印章宽度 | 整数（像素） |
| --height | 是 | 印章高度 | 整数（像素） |
| --x | 是 | 印章 X 坐标 | 整数 |
| --y | 是 | 印章 Y 坐标 | 整数 |
| --output, -o | 否 | 输出文件路径 | 文件路径（默认同目录输出） |

### 需求：多页处理

- 系统应在 PDF 的每一页相同位置添加印章
- 印章位置基于每页的左下角为原点 (0, 0)

### 需求：输出处理

- 默认在原 PDF 同目录下生成新文件，文件名添加 `_sealed` 后缀
- 支持通过 --output 指定输出路径
