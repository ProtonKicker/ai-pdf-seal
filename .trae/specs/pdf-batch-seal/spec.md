# PDF 批量盖章功能规范

## 为什么需要此功能

当前工具只支持单个 PDF 文件盖章，用户需要手动对每个文件执行命令。为提高效率，需要支持目录批量处理，并自动跳过已盖章的文件。

## 变更内容

- 新增目录扫描功能：自动发现目录下的所有 PDF 文件
- 新增已盖章检测功能：通过检测是否存在 `_sealed` 后缀的文件判断是否已处理
- 批量处理：自动对未盖章的 PDF 进行盖章处理

## 影响范围

- 受影响的功能：PDF 批量盖章
- 受影响的代码：CLI 接口 (main.py)

## 新增需求

### 需求：批量处理模式

系统应支持批量处理目录下所有未盖章的 PDF 文件。

#### 场景：批量处理目录

- **当** 用户指定目录路径（而非单个 PDF 文件）
- **则** 扫描目录下所有 .pdf 文件，过滤掉已盖章的文件，对剩余文件批量盖章

### 需求：已盖章检测

系统应通过文件名判断 PDF 是否已盖章。

- 判定规则：如果存在 `filename.pdf` 对应的 `filename_sealed.pdf` 文件，则认为已盖章
- 扫描时应忽略文件名已包含 `_sealed` 的文件

### 需求：命令行参数扩展

| 参数 | 必填 | 说明 | 格式 |
|------|------|------|------|
| --dir, -d | 否 | 目录路径（批量处理模式） | 目录路径 |

### 需求：处理输出

- 批量处理时显示处理进度
- 显示：总文件数、已跳过数、已处理数
- 处理完成后显示汇总信息
