# 配置文件功能规范

## 为什么需要此功能

当前程序所有参数都需要通过命令行指定，使用不便。增加配置文件支持后，用户可以将常用参数保存到配置文件中，程序启动时自动读取，减少重复输入。

## 变更内容

- 新增配置文件支持（默认 config.yaml）
- 程序启动时优先使用命令行参数，其次使用配置文件参数
- 支持通过 --config 参数指定配置文件路径

## 影响范围

- 受影响的功能：所有运行模式
- 受影响的代码：main.py

## 新增需求

### 需求：配置文件格式

采用 YAML 格式，示例：
```yaml
# 印章图片配置
image: stamp.png
width: 50
height: 50
x: 100
y: 100

# 批量处理配置
force: false
```

### 需求：配置文件读取逻辑

- 默认配置文件路径：程序运行目录下的 `config.yaml`
- 支持通过 `--config` 参数指定配置文件路径
- 命令行参数优先级高于配置文件（命令行指定则使用命令行值）

### 需求：命令行参数扩展

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| --config | 否 | 配置文件路径 | config.yaml |
